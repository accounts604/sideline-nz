You don’t *need* Jerseybird-level “icons + examples + super-specific questions” to make the form work… but if you’re trying to **match their conversion + reduce back-and-forth**, then **yes — you want some of it**.

Here’s the truth:

* **Too shallow** → you’ll spend your life in DMs/email clarifying basics (Mikey/Daniel hate this).
* **Too deep** → people bounce (especially on mobile, time-poor admins).
* The win is: **depth in the *right* places**, with **visual guidance** that makes it *feel* easy.

## What Jerseybird is *actually* doing with icons/examples

1. **Reduces cognitive load** (people understand faster than reading)
2. **Increases confidence** (“ah yep, that’s what I need”)
3. **Improves data quality** (you get the details without 10 follow-up questions)
4. **Creates perceived professionalism** (trust signal)

So yes: add icons + examples — but don’t turn it into a 40-question interrogation.

---

# The “Depth” you should add (high ROI, low friction)

## 1) Visual cards for “What do you need?”

Instead of plain pill text, use **cards with icons + 1-liners**:

* 🏉 **Full Playing Kit** — Jerseys, shorts, socks (match-day)
* 🧥 **Training Gear** — tees, singlets, hoodies, trackies
* 🧢 **Supporter Gear** — tees, hoodies, caps, flags
* 👕 **Off-field Apparel** — polos, tees, jackets (staff/community)
* 🤷 **Not sure yet** — we’ll guide you

This is 100% worth doing.

## 2) Ask *one* “specific detail” per category (conditional)

Only ask more when they pick something.

Example conditional questions:

* If **Full Playing Kit** selected:

  * **Garment types** (multi): Jersey, Shorts, Socks
  * **Personalisation** (multi): Names, Numbers, Sponsor logos
* If **Supporter Gear**:

  * **Audience**: Players, Parents, Alumni, Wider community
* If **School** user_type:

  * **Need SLT-friendly design?** (Yes/No)
  * **Delivery deadline aligned to event** (text/date)

This is how you get Jerseybird “depth” without the drop-off.

## 3) Examples / inspiration prompts (tiny but powerful)

Add a section:

* “Upload an example you like” (optional)
* “Or pick a style” with 3 cards:

  * Clean/Classic
  * Bold/Modern
  * Cultural/Heritage

Even if they don’t upload, the selection helps your designer and your quote.

### 3) Add a “CardSelect” component (paste under TogglePill)

```tsx
function CardSelect({
  items,
  selected,
  toggle,
}: {
  items: { label: string; desc: string; icon: string }[];
  selected: string[];
  toggle: (label: string) => void;
}) {
  return (
    <div className="grid grid-cols-1 sm:grid-cols-2 gap-3">
      {items.map((it) => {
        const active = selected.includes(it.label);
        return (
          <button
            key={it.label}
            type="button"
            onClick={() => toggle(it.label)}
            className={cx(
              'text-left rounded-2xl border p-4 transition',
              active ? 'border-black bg-black text-white' : 'border-gray-200 hover:border-black bg-white'
            )}
          >
            <div className="flex items-start gap-3">
              <div className="text-2xl">{it.icon}</div>
              <div>
                <div className="font-medium">{it.label}</div>
                <div className={cx('text-sm', active ? 'text-gray-200' : 'text-gray-600')}>
                  {it.desc}
                </div>
              </div>
            </div>
          </button>
        );
      })}
    </div>
  );
}
```

<div className="space-y-5">
