Below is a **Replit-ready** “Jerseybird-style” **multi-step Start a Project form** with:

* Step-by-step logic (branching for **club vs school**)
* Field validation
* **Progress indicator**
* “Review + Submit” step
* Submits to a **Next.js API route** that forwards to your **GHL Webhook**
* Optional **file upload** pattern (clean + realistic): upload files to Cloudinary (or Uploadcare) → send **URLs** to GHL

> Why URLs? GHL webhooks are great for structured data, but *binary file uploads* are messy unless you embed a native GHL form. URL upload is the most reliable pattern for a custom Next.js form.

---

# 1) Form logic (Jerseybird-style steps)

### Step 1 — Who are you?

* `user_type` (club/school/other)
* `role` (changes labels/options based on user_type)
* `organization`
* `sports` (multi)

**Next button enabled when:** `user_type`, `organization`, `sports` are set.

---

### Step 2 — What do you need?

* `needs` (multi)
* `estimated_quantity`
* `teams_involved` (optional multi)

**Next enabled when:** `needs`, `estimated_quantity`

---

### Step 3 — Timeline + urgency

* `timing`
* `urgency`

**Next enabled when:** both set

---

### Step 4 — Design + budget

* `design_stage`
* `budget_range`
* `files` (optional uploads)
* `notes` (optional)

**Next enabled when:** `design_stage`, `budget_range` (files optional)

---

### Step 5 — Approval + concerns

* `approval_process`
* `main_concern` (multi)

**Next enabled when:** `approval_process` (concern optional)

---

### Step 6 — Contact details + Review

* `name`, `email`, `phone`
* Show review summary
* Submit

**Submit enabled when:** valid contact fields

```tsx
function MultiSelectPills({
  options,
  value,
  onChange,

/**
 * OPTIONAL: file upload pattern (Cloudinary upload preset)
 * - This calls /api/upload which you’ll wire to Cloudinary (or Uploadcare).
 * - On success, we store URLs in form.file_urls.
 */
async function uploadFiles(files: FileList): Promise<string[]> {
  const uploaded: string[] = [];
  for (const file of Array.from(files)) {
    const form = new FormData();
    form.append('file', file);
    const res = await fetch('/api/upload', { method: 'POST', body: form });
    if (!res.ok) throw new Error('Upload failed');
    const data = await res.json();
    uploaded.push(data.url);
  }
  return uploaded;
}
```

## B) `/app/api/ghl/submit-project/route.ts` (forwards to your GHL webhook)

Create this file:

```ts
import { NextResponse } from 'next/server';

export async function POST(req: Request) {
  try {
    const payload = await req.json();

    const webhookUrl = process.env.GHL_PROJECT_WEBHOOK_URL;
    if (!webhookUrl) {
      return NextResponse.json({ error: 'Missing GHL_PROJECT_WEBHOOK_URL' }, { status: 500 });
    }

    // Add any hidden tracking context here (utm, source, page, etc.)
    const enriched = {
      ...payload,
      source: 'sidelinenz.com start-a-project',
      submitted_at: new Date().toISOString(),
    };

    const res = await fetch(webhookUrl, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(enriched),
    });

    if (!res.ok) {
      const text = await res.text().catch(() => '');
      return NextResponse.json({ error: `GHL webhook error: ${res.status} ${text}` }, { status: 502 });
    }

    return NextResponse.json({ ok: true, id: crypto.randomUUID() });
  } catch (e: any) {
    return NextResponse.json({ error: e.message || 'Server error' }, { status: 500 });
  }
}
```

This is a **placeholder**. Choose ONE option:

### Option 1 (best): Cloudinary upload (recommended)

* You’ll set:

  * `CLOUDINARY_CLOUD_NAME`
  * `CLOUDINARY_UPLOAD_PRESET` (unsigned preset)

Create:

```ts
import { NextResponse } from 'next/server';

export async function POST(req: Request) {
  try {
    const form = await req.formData();
    const file = form.get('file') as File | null;
    if (!file) return NextResponse.json({ error: 'No file' }, { status: 400 });

    const cloud = process.env.CLOUDINARY_CLOUD_NAME;
    const preset = process.env.CLOUDINARY_UPLOAD_PRESET;
    if (!cloud || !preset) {
      return NextResponse.json({ error: 'Missing Cloudinary env vars' }, { status: 500 });
    }

    const cloudinaryForm = new FormData();
    cloudinaryForm.append('file', file);
    cloudinaryForm.append('upload_preset', preset);

    const uploadRes = await fetch(`https://api.cloudinary.com/v1_1/${cloud}/auto/upload`, {
      method: 'POST',
      body: cloudinaryForm,
    });

    if (!uploadRes.ok) {
      const text = await uploadRes.text().catch(() => '');
      return NextResponse.json({ error: `Cloudinary error: ${uploadRes.status} ${text}` }, { status: 502 });
    }

    const data = await uploadRes.json();
    return NextResponse.json({ url: data.secure_url });
  } catch (e: any) {
    return NextResponse.json({ error: e.message || 'Upload server error' }, { status: 500 });
  }
}
```

# 4) How this maps to GHL

In GHL you’ll do one of these patterns:

### Pattern A (simple): Webhook → Workflow → Create/Update Contact + Opportunity

* Trigger: **Inbound Webhook**
* Actions:

  * Create/Update Contact (use `email` / `phone`)
  * Add tags (use `user_type`, `sports`, `needs`, etc.)
  * Create Opportunity in “Custom Orders” pipeline
  * Internal notification (email/slack)
  * Auto-reply to lead (email/sms)

### Pattern B (more “Jerseybird”): Route different workflows based on inputs

* If `user_type = school` → school workflow tone + SLT-friendly pack
* If `urgency = High` or `timing = ASAP` → instant “rush” workflow + internal alert

(Your Integration Plan already supports this GHL-first approach.)
